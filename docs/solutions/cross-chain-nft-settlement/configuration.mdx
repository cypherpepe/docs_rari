# Configuration

The component does require some parameters in order to get a basic settlement flow working.
If you do not pass in these parameters, then it will throw an error.
In case you run into errors, the component uses [loglevel](https://www.npmjs.com/package/loglevel) for logging while in development.

## Parameters

Here are some useful parameters for setting up the button.

```
buttonProps.label // the label you want on the button
buttonProps.className // for styling the button
chainList // list of chains that you want to allow fungible tokens from
nftDetails.chainName // network that the NFT resides on
nftDetails.price // how much the NFT should be sold for (in native coin)
nftDetails.id // token id
jsonRPCUrlMap // map of rpcProviders <-> chains, should match the chainList param
```

We have exported types in our package that your editor or development flow can use to discover and validate additional parameters.

## Example Configuration

```javascript
import { RarimoPayButton } from "@rarimo/rarimo-settlement-component";

const chainList = [
  {
    contractAddress: "0x3c16183c1c0e28f1a0cb9f8ee4b21d0db208ca46",
    chainId: 5,
    symbol: "Goerli",
    chainType: "evm",
    icon: "https://google.com/15735.png",
    name: "Goerli",
  },
];

const jsonRPCUrlMap = {
  5: "https://eth-goerli.g.alchemy.com/v2/...",
};

function App() {
  return (
    <RarimoPayButton
      buttonProps={{
        label: "Buy with Rarimo",
      }}
      chainList={chainList}
      nftDetails={{
        chainName: "ethereum",
        price: 0.03,
        id: "123",
      }}
      jsonRPCUrlMap={jsonRPCUrlMap}
    />
  );
}

export default App;
```
